# 概要

新ツール・アーキテクチャ等の検証用 Laravel です。

ツールを試す際、Public リポジトリでは制限なし・Private では制限あり（有料のみ）といったケースが多いため、あえて **Public リポジトリ** にしています。

## 環境

| 項目   | バージョン |
|--------|------------|
| Laravel | 12        |
| PHP     | 8.5       |

## 使い方

### セットアップ

```bash
herd composer install
cp .env.example .env
herd php artisan key:generate
herd php artisan migrate
```

### URL

**https://laravel-playground.test**

### 主な操作

| 操作           | 画面・手順 |
|----------------|------------|
| タスク一覧     | トップ（`/`）で表示。期限が近い順。完了タスクは一覧の下に表示される |
| 新規タスク作成 | タイトル・詳細・期限日を入力して POST 送信 |
| ステータス変更 | 一覧の各タスクから「未着手 / 進行中 / 完了」を選択して更新 |
| 削除（論理削除） | 一覧の削除ボタンから実行 |

期限切れのタスクは一覧で赤文字で表示される。

## ブランチ運用

- **master**: 基本的な MVC 構成の簡易タスク管理アプリ。検証用の追加は入れない。
- **xxx_master**: 検証したいアーキテクチャやツールを試す用に、master から分岐して作成する。検証終了後も残しておいてOK。
- **master へのマージ**: 開発ルールの変更を反映するとき、またはmasterブランチでないと確認できない場合などに限定。通常の検証結果の反映は master にはマージしない。特にアーキテクチャは master（基本的なMVC）と比較することで、理解が深まるため。

### このブランチで検証しているものをここに記載

- **ブランチ名**: master
- **検証ツール**:
  - [CodeRabbit](https://www.coderabbit.ai/ja)
    - コードレビューの自動化
    - サンプル（[タスクのステータス変更操作と一覧への反映を実装する #19](https://github.com/pasgroup/laravel-playground/pull/22)）
      - 重要な指摘は該当のコードへのコメント（修正案まで出してくれる）
      - 細かなアドバイスについては、コメントのNitpick commentsを開くと見られる
  - [Codecov](https://app.codecov.io/gh/pasgroup/laravel-playground/)
    - カバレッジ表示
    - Github actions でテストを実行して反映させている
    - パーセンテージも検証しているので、Github actions の実行時に規程を下回った場合に、actionを失敗させたりもできる
    - ブランチごとにカバレッジの確認が可能

- **採用・検証中のアーキテクチャ**: 
  - 基本的なMVC（ここからブランチを切って、他のアーキテクチャを検証しやすくするため）
  - テストコードなどは現在のルールを踏襲

## 開発要件

### 簡易タスク管理アプリ（ToDo＋期限付き）

**タスクの構成**

- タイトル・詳細・期限日・ステータス（未着手 / 進行中 / 完了）

**機能**

- POST 送信のみ受け付ける
- DB へ保存する
- 一覧画面で表示する（期限が近い順）
- ステータスを変更できる（更新処理）
- 論理削除可能

**表示**

- 完了したタスクは一覧の下に表示する
- 期限切れのタスクは赤文字で表示する
- UI は自由
